## 1. Project Overview

The Doomlings Upseller App is a custom-built Shopify embedded app developed for Doomlings LLC's Shopify Plus DTC store. It replaces a collection of third-party apps (upsell, BXGY, post-purchase, countdown, cursor) with a single, fully owned system — eliminating recurring SaaS fees and giving Doomlings complete control over storefront behavior and revenue optimization tools.

The app was built in approximately 16 days (March 22 – April 6, 2026), with approximately 80-85% of production code generated by Claude. Developer effort covered infrastructure provisioning, Shopify app registration, and deployment.

### Core Goals

- Increase average order value through upsell recommendations, bundle offers, and post-purchase offers
- Automate gift-with-purchase and Buy X Get Y promotions at the cart level
- Provide Doomlings staff with a single admin interface for all promotional rules
- Give the storefront team theme scheduling, geo-targeted campaigns, and custom cursor tools without requiring developer involvement

## 2. System Components

The app consists of five interconnected components.

### Component 1 — Admin Dashboard (Embedded)

Shopify-embedded admin panel accessible from the Shopify Admin sidebar. Protected by Shopify App Bridge session tokens.

- **Overview tab** — daily order count, total revenue, average order value, upsell-attributed revenue; 7/30/90-day selectors; auto-refreshes every 60 seconds
- **Products tab** — searchable product catalog browser with variant details and storefront/admin deep links
- **Cart Limits tab** — configure per-product maximum purchase quantities enforced at the cart level
- **Upsells tab** — create trigger-based product recommendations (when customer adds Product A, suggest Product B)
- **Buy X Get Y tab** — configure automated gift rules with auto-add, priority ordering, and per-order caps
- **Bundle Offers tab** — multi-product bundle configurations with generated discount codes and storefront display
- **Geo Countdown tab** — time-limited countdown campaigns targeted by country and page type
- **Custom Cursor tab** — branded cursor themes (glow, stardust, doomlings, retro) with hover/click effects
- **Theme Scheduler (Launchpad) tab** — schedule automated theme publishing to specific timezones
- **Post-Purchase tab** — configure post-checkout upsell offers shown immediately after order completion
- **Stats tab** — per-rule conversion analytics for BXGY and post-purchase offers (views, accepts, revenue)

### Component 2 — Standalone Dashboard

An alternative admin portal accessible outside of the Shopify Admin iframe, authenticated via secure session cookie. Mirrors all functionality of the embedded dashboard. Useful for staff who need access without navigating through Shopify Admin.

- Route: `/dashboard/[tab]`
- Auth: JWT-signed cookie (`SESSION_SECRET`), validated by Next.js middleware
- All data via `/api/standalone/*` endpoints

### Component 3 — Theme App Extension (Storefront Widgets)

Nine Liquid/JavaScript app blocks delivered via Shopify's theme app extension framework. Compatible with all Online Store 2.0 themes including Pebble (FoxEcom). No theme file edits required — blocks survive theme updates automatically.

See Section 6 for full block inventory.

### Component 4 — Shopify Functions (Cart Discount)

A WebAssembly cart transform function that applies line-item and delivery discounts in the Shopify checkout pipeline. Reads configuration from a shop metafield (`upsale.config`) and handles:

- BXGY automatic gift line injection
- Bundle offer discount application
- Delivery discount generation

### Component 5 — Post-Purchase Extension

A Shopify checkout post-purchase UI extension that intercepts the order confirmation page to present a one-click upsell offer. Integrates with the app backend for offer resolution and signed changeset generation.

### 5.2 Registered Webhooks

| Topic | Handler |
|---|---|
| `orders/create` | Increment daily order count + revenue in Firestore; attribute upsell revenue from order properties |
| `orders/cancelled` | Decrement order count + revenue |
| `app/uninstalled` | Mark shop uninstalled, purge all sessions |
| `customers/data_request` | GDPR — acknowledge data request |
| `customers/redact` | GDPR — delete customer PII |
| `shop/redact` | GDPR — delete all shop data 48h post-uninstall |

All webhooks are HMAC-verified before processing. Handler always returns `200 OK` for valid requests to prevent Shopify retries; errors are logged internally.

## 6. Theme App Extension Blocks

All blocks are delivered via the `upsell-widget` theme app extension and are installable through the Shopify theme editor without code changes.

| Block | Placement | Purpose |
|---|---|---|
| `upsell.liquid` | Product pages | Trigger-based product recommendations; polls `/api/public/gift-rules` |
| `bundle-offers.liquid` | Product / cart pages | Bundle selection UI with discount code generation and price breakdown |
| `gift-notification.liquid` | Global (floating) | Toast notification when BXGY gift is auto-added to cart |
| `cart-limits-guard.liquid` | Cart page | Enforces max-quantity rules, prevents checkout if exceeded |
| `cart-upsell.liquid` | Cart page | Cart-context upsell recommendations based on current line items |
| `geo-countdown.liquid` | Any page | Geo-targeted countdown timer; matches by country + page type |
| `custom-cursor.liquid` | Global | Replaces browser cursor with branded theme; scoped by page target |
| `star_rating.liquid` | Product pages | Star rating display using `stars.liquid` snippet |

**Block integration pattern:** Each block renders a root container with `data-shop`, `data-api-base`, and context attributes. Inline JavaScript reads these attributes, fetches from the corresponding `/api/public/*` endpoint, and renders the UI client-side. Blocks handle `401` (shop not installed) and error states gracefully with no visible failure.

## 8. Shopify Extensions Detail

### 8.1 Post-Purchase Extension

- **Extension point**: `Checkout::PostPurchase::ShouldRender` + `Checkout::PostPurchase::Render`
- **Flow**:
  1. `ShouldRender` calls `/api/post-purchase/offer` with JWT — backend queries Firestore for matching offer
  2. If offer found: `Render` displays product image, price, shipping/tax estimate, accept/decline buttons
  3. On accept: calls `/api/post-purchase/sign-changeset` to get a signed token
  4. Calls `applyChangeset()` to add the item to the existing order without a new checkout
- **Security**: JWT signed with `SESSION_SECRET`; changeset signed with Shopify partner credentials

### 8.2 Cart Discount Function (WebAssembly)

- **Targets**: `cart.lines.discounts.generate.run` and `cart.delivery-options.discounts.generate.run`
- **Config source**: `upsale.config` shop metafield (written by app backend on rule save)
- **Capabilities**: Line-item percentage discounts, flat discounts, free-item injection, delivery discounts
- **Test fixtures**: Provided for discount, no-discount, order-only, product-only, and delivery-option scenarios

## 9. Security

| Concern | Approach |
|---|---|
| Shopify iframe embedding | CSP `frame-ancestors https://*.myshopify.com https://admin.shopify.com` |
| Webhook authenticity | HMAC verified on every inbound webhook before processing |
| OAuth CSRF | State parameter validated on callback |
| Admin API access | App Bridge JWT extracted and validated per request |
| Standalone session | JWT-signed cookie; `verifyShop()` on every protected route via middleware |
| Public API abuse | All `/api/public/*` endpoints verify shop is installed before responding |
| Firestore access | Server-side only; no client-side Firebase credentials exposed |
| Multipass / secrets | Not applicable to this app (no customer auth layer) |

## 12. Known Limitations & Future Considerations

| Item | Notes |
|---|---|
| GDPR handlers | `customers/redact` and `shop/redact` webhooks are acknowledged but data deletion is logged only — not yet fully implemented. Required for Shopify app store compliance. |
| Launchpad cron reliability | Currently dependent on an external cron service (EasyCron). Consider moving to Vercel Cron Jobs for reliability. |
| Analytics attribution | Upsell revenue is attributed via order `properties` set by storefront widgets. If a customer disables JavaScript, attribution will not fire. |
| Cart function limits | Shopify Functions have a 5ms execution budget. Complex rule sets with many conditions should be profiled against this limit. |
| No customer-facing auth layer | This app has no persistent customer login or loyalty/points system. The Doomling Loyalty & Reviews spec (v2.0, April 2026) covers that separately. |
| Firebase — no migration path documented | If the app needs to scale beyond Firebase's free tier, a migration to Postgres (Supabase/Neon) should be planned. |
